# Телефонный справочник

Консольное приложение на Python для ведения простого телефонного справочника. Контакты хранятся в JSON-файле, а работа с ними выполняется через интерактивное меню в терминале.

Проект написан без внешних зависимостей и использует только стандартную библиотеку Python.

## Возможности

- загрузка контактов из JSON-файла;
- автоматическое создание файла контактов, если он отсутствует;
- просмотр контактов в табличном виде;
- пагинация списка по 10 контактов на страницу;
- поиск по имени, телефону и комментарию;
- добавление новых контактов;
- выбор контакта по ID;
- редактирование имени, телефона и комментария;
- удаление контактов;
- сохранение изменений в JSON-файл;
- предупреждение о несохраненных изменениях при выходе;
- валидация структуры JSON и данных контактов.

## Требования

- Python 3.10 или новее.

Установка зависимостей не требуется.

## Запуск

```bash
python main.py
```

При запуске приложение запросит путь к файлу контактов:

```text
Файл контактов [contacts.json]:
```

Можно нажать `Enter`, чтобы использовать файл по умолчанию `contacts.json`, или указать свой путь.

Правила обработки пути:

- если файл не существует, он будет создан автоматически;
- если указать путь без расширения, приложение добавит расширение `.json`;
- если указать расширение, отличное от `.json`, приложение завершит загрузку с ошибкой;
- если указать путь с несуществующими папками, они будут созданы автоматически.

Примеры:

```text
contacts
data/my_contacts.json
```

## Главное меню

После загрузки данных доступны действия:

1. `Показать все контакты` - открыть список контактов с пагинацией.
2. `Найти контакт` - найти контакты по имени, телефону или комментарию.
3. `Добавить контакт` - создать новый контакт.
4. `Выбрать контакт` - открыть контакт по ID.
5. `Сохранить` - записать текущие изменения в JSON-файл.
6. `Выход` - завершить работу приложения.

В списке контактов можно перейти на следующую или предыдущую страницу, выбрать конкретную страницу, открыть контакт по ID или вернуться назад.

В карточке выбранного контакта можно:

- отредактировать поля контакта;
- удалить контакт;
- вернуться назад.

## Формат данных

Контакты хранятся в JSON-файле в виде массива объектов:

```json
[
  {
    "id": 1,
    "name": "Ivan Petrov",
    "phone": "+79991234567",
    "comment": "Test contact"
  }
]
```

Поля контакта:

- `id` - положительное целое число, уникальный идентификатор контакта;
- `name` - непустая строка с именем;
- `phone` - непустая строка с телефоном;
- `comment` - строка с комментарием, может быть пустой.

При добавлении нового контакта ID назначается автоматически: берется максимальный существующий ID и увеличивается на 1.

## Сохранение

Добавление, редактирование и удаление контактов меняют данные в памяти приложения. Чтобы записать изменения в файл, выберите пункт `Сохранить`.

При выходе приложение проверяет наличие несохраненных изменений и предлагает сохранить их.

Сохранение выполняется через временный файл `*.json.tmp`, который затем заменяет основной JSON-файл.

## Структура проекта

```text
.
├── main.py         # точка входа приложения
├── controller.py   # сценарии работы меню и координация model/view
├── model.py        # контакт, телефонная книга, JSON-хранилище и ошибки
├── view.py         # консольный ввод, вывод и форматирование таблиц
└── README.md
```

## Архитектура

Код разделен на несколько слоев:

- `model.py` отвечает за данные, валидацию контактов, поиск, добавление, редактирование, удаление и чтение/запись JSON;
- `view.py` отвечает за консольный интерфейс, ввод пользователя и вывод таблиц;
- `controller.py` связывает модель и представление, управляет меню и сценариями работы;
- `main.py` создает объекты приложения и запускает контроллер.

## Ограничения и ошибки

- Файл контактов должен быть JSON-массивом.
- У каждого контакта должны быть поля `id`, `name` и `phone`.
- В JSON-файле не должно быть дубликатов по `id`.
- `id` должен быть положительным целым числом.
- `name` и `phone` не могут быть пустыми строками.
- Если JSON поврежден или данные контактов некорректны, приложение завершит загрузку с сообщением об ошибке.
